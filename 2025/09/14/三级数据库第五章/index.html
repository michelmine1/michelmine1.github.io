三级数据库第五章——UML与数据库应用系统

一、UML图

1、图

1、活动图

活动图主要用来描述系统、用例和程序模块中逻辑流程的先后执行次序，并行次序。

基本组成元素：

1. 基本组成元素
元素 描述
起始节点 表示流程的开始点，每个活动图必须有且仅有一个起始节点，用黑色圆点表示
活动节点 表示流程中的一个步骤或操作，用圆角矩形表示
控制流 连接节点，表示活动之间的转换和顺序，用实现箭头表示
结束节点 表示流程的终止，分为活动结束(终止特定流程)和流程结束(终止所有流程)，用双层空心圆表示
2. 决策与合并
元素 描述
决策节点 菱形表示，用于表示条件分支，每个输出流应有监护条件
合并节点 同样使用菱形，用于多个可选路径
3. 并行处理
元素 描述
分叉节点 粗水平线，将单一流程分为多个并发执行的子流程
结合节点 粗水平线，等待所有并行流程完成
4. 其他重要元素
元素 描述
对象节点 表示活动处理的数据或对象
泳道 将活动按职责分组，通常表示不同的参与者或组织单元
信号 表示发送或接收事件，如发送电子邮件或接收中断

*一个活动图只有一个起始点，但可以有多个结束点。

2、状态图

主要用于描述一个对象在其生命周期内所经历的各种状态之间的转换关系，它侧重于对象的动态行为，而非静态结构。

3、通信图

通信图是一种交互图，强调的是对象之间的消息传递关系，属于描述系统动态结构的图

4、顺序图

（1）顺序图也是交互图的一种，它按照时间顺序展示对象之间的交互过程，同样是描述系统动态行为的。

元素 描述 作用
对象 横向矩形框
通信 水平消息线 箭头说明消息的类型（同步、异步、简单）

（2）顺序图可以表示递归过程，当一个操作调用它本身时，消息总是同步的。

​ 在顺序图中可以描述如何创建和销毁对象，创建或销毁一个对象的消息一般是同步的。

​ 顺序图中的消息可以有条件，并规定只有条件为真的时候才可以发送和接收消息，如果条件之间不互相排斥，则消息可能会并行发出。

5、类图

类图是UML中用于描述系统静态结构的核心图之一，它展示了系统中的类、类的属性、类的操作以及类之间的关系（如关联、聚合、组合、继承等），能够清晰地反映系统的静态结构。

包图是表示包及包之间的关系的类图。

组成元素：

元素 描述 作用
类（泛化） 空心三角实线连接 子类对父类关系的继承
接口（实现） 空心三角虚线连接 类似于继承，但是多继承
聚合 空心菱形实线连接 弱拥有关系
合成（组合） 实心菱形实线连接 强拥有关系
关联 普通箭头实线连接 指向被拥有者
依赖 普通箭头虚线连接 指向被使用者

6、协作图

2、UML的语义定义

1. 四层元模型架构

这是理解UML语义的基础框架。它定义了UML语言本身是如何被描述的，确保了UML的自描述性。

层级 描述 例子
M0: 实例层 运行时的具体对象世界。 一个名叫 张三 的具体客户对象，其 age = 25。
M1: 模型层 用户创建的UML模型，描述M0层。 一个Customer类，拥有name: String和age: int属性。
M2: 元模型层 定义M1层模型元素的语言（UML本身）。 Class, Attribute, Operation 等UML元类。
M3: 元-元模型层 定义M2层元模型的语言。最顶层，通常只有一个核心概念。 MetaClass, MetaAttribute (例如，MOF - Meta-Object Facility)。

关系：M3层定义M2层，M2层定义M1层，M1层描述M0层。每一层都是下一层的实例。
